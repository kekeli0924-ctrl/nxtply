## Role-isolation check

| Route | Auth guard | Ownership check |
|---|---|---|
| `POST /api/blocks` | `requireCoach` | roster membership verified via `coach_players` |
| `GET /api/blocks` | `requireCoach` | `WHERE coach_id = req.userId` |
| `GET /api/blocks/:id` | `requireCoach` | `row.coach_id !== req.userId` → 403 |
| `GET /api/blocks/:id/export` | `requireCoach` | `row.coach_id !== req.userId` → 403 |
| `GET /api/blocks/active-for-player` | global `authMiddleware` only | player data scoped to `req.userId` |
| `POST /api/blocks/:blockId/digests` | `requireCoach` | `row.coach_id !== req.userId` → 403 |
| `GET /api/blocks/:blockId/digests` | `requireCoach` | `row.coach_id !== req.userId` → 403 |
| `DELETE /api/blocks/:blockId/digests/:id` | `requireCoach` | `digest.coach_id !== req.userId` → 403 |
| `GET /api/public/digest/:slug` | **none** (public by design) | slug is 128-bit random; revoked_at checked |
| `POST /api/public/digest/:slug/opened` | **none** (telemetry) | validates slug exists + not revoked before emitting |
| `GET /api/events/metrics` | `requireFounder` (username match) | founder-only |
| `GET /api/events/export` | `requireFounder` | founder-only |
| `POST /api/events` | global `authMiddleware` | user can only emit allowlisted client events |

Three-role isolation is preserved for all new endpoints. No cross-role data leakage found. The public digest surface is intentionally unauthenticated and exposes only a privacy-scrubbed snapshot (no usernames, no raw notes, no parent-only fields — enforced in `buildSnapshot`).
