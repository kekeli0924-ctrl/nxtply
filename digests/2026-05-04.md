## Video AI / Multimodal

### Timestamped event extraction — direct upgrade path

Composed's `server/services/videoAnalyzer.js` uses Gemini to extract aggregate stats (shots taken, pass accuracy, technique quality). Gemini 3 Flash supports temporal event localisation with MM:SS timestamps. Upgrading the prompt to request a structured event log — `[02:14] shot, right foot, on target` — would let the app auto-populate stats per drill block rather than per session. This is a one-file change (prompt + response parser) with high product value.

### On-device pose estimation

[MediaPipe Pose Landmarker](https://ai.google.dev/edge/mediapipe/solutions/vision/pose_landmarker) runs in-browser via WASM at 15–30 FPS on modern phones (33 keypoints). It could power a real-time technique overlay during live recording in `src/components/CameraSetup.jsx` — hip rotation and knee alignment during a shooting drill — without any server round-trip. Constraint: MediaPipe (~6MB WASM) and FFmpeg.wasm (~25MB WASM) both compete for memory on mobile; only load on demand and never simultaneously.

### Sports video AI research highlights

- **[I3D + TSN + pose fusion](https://www.sciencedirect.com/science/article/abs/pii/S1746809425008675)**: paper combining I3D and Temporal Segment Networks with multi-stage pose-feature fusion for improved action detection on SoccerNet/Sports-1M.
- **[SAM2 player tracking](https://arxiv.org/html/2604.08722v1)**: Segment Anything 2 seeded from a first-frame object detector enables full-clip segmentation tracking without per-clip retraining.
- **[YOLOv11 soccer analytics](https://scholarworks.uark.edu/dtscuht/25/)**: single-shot detection at 60 FPS for real-time analytics.
- **[PassNet](https://github.com/jonpappalord/PassNet)** remains the reference architecture for pass/shot event classification; extensible to fouls and saves.
- WSC Sports reports >98% event-detection accuracy on highlight auto-generation including near-misses and saves — the quality bar server-side Gemini calls should target.

### Model selection guide for video

| Task | Best model today |
|------|-----------------|
| Timestamped shot/pass events from training footage | Gemini 3 Flash (native video, MM:SS temporal, 10 videos/request) |
| Per-frame technique analysis (form, foot contact) | Claude Opus 4.7 (3× higher vision resolution, image-per-frame) |
| Multimodal session search / semantic recall | Gemini Embedding 2 (unified text+video embedding space) |

## Browser / PWA Capabilities

### WebGPU — production baseline

[WebGPU reached Baseline in January 2026](https://www.utsubo.com/blog/frontier-web-apis-2026-production-ready): Chrome, Edge, Firefox, and **Safari 26** (full WebGPU on Metal, iOS 26+) all ship it stable. [~70–75% of mobile users](https://appdevelopermagazine.com/webgpu-in-ios-26/) can run WebGPU. Benchmarks show [3–5× over WebGL and 10–15× over WASM-only](https://www.buildmvpfast.com/blog/webgpu-browser-ai-inference-cost-savings-2026) for transformer workloads — enables MediaPipe Pose Landmarker at interactive frame rates in `src/components/CameraSetup.jsx`.

**Caveat**: iOS 26.3 Safari + ONNX Runtime Web has a documented [crash after ~500 inference iterations](https://github.com/microsoft/onnxruntime/issues/27584). Monitor before shipping any on-device inference loop.

### OPFS — high-performance in-browser storage

[Origin Private File System](https://developer.mozilla.org/en-US/docs/Web/API/File_System_API/Origin_private_file_system): synchronous Worker API for large sequential I/O, supported on Chrome, Firefox, and iOS 17.4+. OPFS-backed FFmpeg.wasm intermediate files would reduce peak memory on low-RAM phones during encoding in `src/utils/ffmpeg.js`. Pattern: OPFS for file staging + IndexedDB for metadata + WebGPU for compute = clean local-first video pipeline.

### WebCodecs — near baseline

[WebCodecs](https://developer.mozilla.org/en-US/docs/Web/API/WebCodecs_API) is [fully supported in Safari 26](https://www.utsubo.com/blog/frontier-web-apis-2026-production-ready); Firefox has decode + most encode paths. Production users: Zoom Web, Loom, Adobe Premiere Web. For `src/components/CameraSetup.jsx`, a `MediaRecorder` → `VideoEncoder` → OPFS → tus upload pipeline would eliminate the FFmpeg hang watchdog (`encodeHangRef` at `VideoUpload.jsx:37`) and reduce the 30–60s encode wait to near-real-time.

### MediaRecorder — universal floor

[WebM/VP8/Opus is now supported across all major browsers](https://copyprogramming.com/howto/record-browser-tab-with-javascript), eliminating per-browser codec fragmentation. Safari 18.4 added non-timeslice mode encoding.

### Web Push — missing from Composed

The [Push API](https://developer.mozilla.org/en-US/docs/Web/API/Push_API) + VAPID works for PWAs on Android and iOS 16.4+. There is no push infrastructure anywhere in Composed's codebase. A "you haven't trained today" reminder at 6pm is the highest-leverage missing PWA feature: it drives daily active use more reliably than any UI improvement, and no hardware competitor (Trace, Veo) sends solo training reminders. See Recommended Actions §4.

### WebNN — emerging

[Web Neural Network API](https://www.ddevtools.com/updates/2026-01-webgpu-webnn-browser-ai) joins WebGPU and WASM SIMD as a production-platform API for on-device inference. Not yet relevant to Composed's current stack but relevant to a future on-device analysis path.

## Auth / Data Layer

### JWT — algorithm-confusion CVEs (Composed not directly affected, but audit warranted)

Multiple CVEs landed this week targeting the JWT algorithm-confusion pattern — where the `alg` header from the token itself is trusted instead of enforced server-side:

- **[CVE-2026-34950](https://advisories.gitlab.com/pkg/npm/fast-jwt/CVE-2026-34950/)** (CVSS 9.1): `fast-jwt` (npm) — leading whitespace in the key string defeats the `publicKeyPemMatcher` regex, re-enabling algorithm confusion. Composed uses `jsonwebtoken`, **not** `fast-jwt` — not directly affected.
- **[CVE-2026-22817](https://github.com/honojs/hono/security/advisories/GHSA-f67f-6cw9-8mq4)** (CVSS 8.2): Hono JWT middleware derives the verify algorithm from the token's own `alg` header. Composed uses Express, **not** Hono — not directly affected.

**Verify posture in `server/auth.js`:** Composed's JWT verify calls should pass an explicit `algorithms` whitelist and never trust the token-embedded `alg`. The current implementation uses `jsonwebtoken@9.0.3` — confirm that all `jwt.verify()` calls specify `{ algorithms: ['HS256'] }` (or whichever algorithm was used to sign). This is a 10-minute audit, not a vulnerability response.

### SQLite WAL-reset corruption bug — upgrade now ⚠️

**SQLite 3.51.3 (March 13, 2026)** patched a [15-year-old WAL-reset database corruption bug](https://sqlite.org/releaselog/3_51_3.html) — a WAL-reset during a write transaction could corrupt the database in certain concurrent access patterns. SQLite 3.52.0 and 3.53.0 include the same fix. Composed uses `PRAGMA journal_mode = WAL` in `server/db.js`. The `better-sqlite3@^12.6.2` range resolves to 12.9.0 (bundling SQLite 3.53.0, which includes the fix), but this has not been explicitly deployed. See Recommended Actions §5.

### better-auth — worth watching for email gap

[`better-auth`](https://www.better-auth.com/) (v1.x) is gaining traction as an all-in-one auth library for Node + SQLite. Migrating from `server/auth.js` is not worthwhile — the existing implementation is solid — but `better-auth`'s email plugin bundles transactional email (password reset, verification) that Composed entirely lacks.

### Drizzle ORM — on radar

[Drizzle](https://orm.drizzle.team/) is maturing as the lightweight TypeScript ORM for SQLite + better-sqlite3 stacks. Not worth migrating now, but the multi-join queries in `server/routes/roster.js` and `server/routes/assignedPlans.js` are natural first candidates if query complexity grows.

### Passkeys — adoption inflection

87% of enterprises are actively deploying or piloting FIDO2 passkeys ([Security Boulevard](https://securityboulevard.com/2026/04/8-reasons-87-of-enterprises-deploying-passkeys-in-2026/)). OpenAI launched passwordless-only sign-in for ChatGPT on May 4. No immediate action for Composed, but `server/auth.js` supporting a WebAuthn credential alongside the current Google OAuth path would align with the industry direction and reduce the password-reset gap.

## Recommended Actions

1. **Audit Zod 4.4.0 breaking changes in `server/validation.js`** — create a branch, run `npm install zod@4.4.3`, then `npm test`. Check: (a) any `.merge()` calls on schemas with `.refine()` or `.transform()` — replace with `.extend()`; (b) any `z.undefined()` properties — add `.optional()`. The 16 schemas are all in one file; the audit is a 30-minute task. Bump the pin from `^4.3.6` to `^4.4.3` in `package.json`.

2. **Upgrade the Gemini video prompt in `server/services/videoAnalyzer.js` to Gemini 3 Flash + timestamped events** — update the model identifier from `gemini-2.5-flash` (or equivalent) to `gemini-3-flash` and change the system prompt from aggregate stats to a structured event log (`[MM:SS] event_type, detail`). Update the response parser to populate stats per time-block. Also bump `@google/genai` to `^1.51.0` in `package.json` and audit any code that sets `audio.rate` — rename to `audio.sample_rate`. Hold any major architecture changes until after Google I/O (May 19–20).

3. **Replace `react-webcam` in `src/components/CameraSetup.jsx`** — the library is abandoned (no release since 2022). A 60–80 line `getUserMedia` + `MediaRecorder` wrapper in `src/utils/camera.js` would remove the dependency, add WebCodecs-based encoding support, and eliminate a long-tail iOS 18+ compatibility risk. `react-camera-pro` is an active third-party alternative if preferred.

4. **Implement Web Push for training reminders** — (a) `npx web-push generate-vapid-keys` and add keys to `.env`; (b) add a `push_subscriptions` table in `server/db.js` with a `POST /api/push/subscribe` route; (c) add a nightly job that queries users who haven't logged a session today and fires a VAPID push. A "You haven't trained today" push at 6pm converts idle-app users to active ones — no hardware competitor can replicate this for solo training.

5. **Confirm SQLite WAL fix is deployed — bump `better-sqlite3` pin to `^12.9.0`** — run `npm install` on staging and verify that `better-sqlite3` resolves to 12.9.0 (SQLite 3.53.0, which contains the WAL-reset corruption fix). Make the lower bound explicit in `package.json` so CI never silently resolves to an older version. Then audit all `jwt.verify()` calls in `server/auth.js` to confirm they pass an explicit `algorithms` whitelist — a 10-minute hardening task surfaced by this week's CVEs.
